packages/tsconfig/package.json 설명

이 파일은 코드를 실행하는 파일이 아니라, packages/tsconfig 폴더를 하나의 패키지로 인식시키는 설명서다.

내용:

{
  "name": "@atelier/tsconfig",
  "version": "0.1.0",
  "private": true,
  "files": [
    "base.json",
    "next-app.json"
  ]
}

1. name

"name": "@atelier/tsconfig"

이 패키지의 이름이다.

@atelier/tsconfig는 "Atelier 프로젝트에서 쓰는 TypeScript 설정 패키지"라는 뜻으로 보면 된다.

나중에 다른 앱이나 패키지에서 이런 식으로 참조할 수도 있다.

{
  "extends": "@atelier/tsconfig/next-app.json"
}

다만 지금은 아직 그렇게 하지 않고, 상대 경로로 참조하고 있다.

{
  "extends": "../../packages/tsconfig/next-app.json"
}

지금 상대 경로를 쓴 이유는 의존성 연결이나 lockfile 변경 없이 바로 쓸 수 있기 때문이다.

2. version

"version": "0.1.0"

패키지 버전이다.

실제로 npm에 배포하는 패키지는 아니지만, 워크스페이스 안에서도 패키지 형태를 갖추려면 보통 버전을 둔다.

지금 의미는 "아직 초기 설정 패키지다" 정도로 보면 된다.

3. private

"private": true

이 패키지를 외부 npm registry에 실수로 배포하지 못하게 막는 설정이다.

packages/tsconfig는 모노레포 내부에서만 쓰는 설정 패키지다.
누가 실수로 pnpm publish 같은 명령을 실행해도 배포되지 않게 막아둔 것이다.

4. files

"files": [
  "base.json",
  "next-app.json"
]

이 패키지에서 의미 있는 파일은 이 두 개라고 표시하는 목록이다.

base.json은 공통 TypeScript 설정이다.
next-app.json은 Next.js 앱용 TypeScript 설정이다.

지금은 private: true라 실제 배포는 안 하지만, 이 목록을 써두면 패키지의 목적이 명확해진다.

폴더 구조로 보면 다음과 같다.

packages/tsconfig/
  package.json   -> 이 폴더는 "공통 TS 설정 패키지"라고 이름 붙이는 명찰
  base.json      -> 모든 곳에서 공통으로 쓸 기본 규칙
  next-app.json  -> Next 앱이 쓸 추가 규칙

결론:

package.json 자체가 TypeScript 설정을 직접 하지는 않는다.

대신 packages/tsconfig 폴더를 그냥 json 파일 모음이 아니라, @atelier/tsconfig 라는 워크스페이스 패키지로 만들어주는 역할을 한다.

즉 이 파일의 핵심은:

packages/tsconfig를 모노레포 안의 공식 공통 설정 패키지로 등록하는 파일이다.
