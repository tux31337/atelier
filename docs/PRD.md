# PRD: Atelier (블로그 MVP)

## 목표
한 명의 작업자(블로그 운영자)가 자기 글과 작업물을 정리해서 발행하는 개인 블로그를 만든다. 같은 워크스페이스 위에서 이후 추가될 프론트엔드 프로젝트도 같은 셋업으로 굴러갈 수 있도록 모노레포로 구성한다.

## 사용자
- **운영자(본인)** — 글을 쓰고 게시하는 단일 저자. 글은 로컬 에디터에서 마크다운/MDX로 작성한 뒤 PR/푸시로 배포한다.
- **방문자** — 글을 읽으러 오는 익명 독자. 모바일·데스크톱·검색엔진 봇이 주된 환경.

## 핵심 기능 (MVP 범위)
1. **글 목록 페이지** (`/`) — 발행일 기준 최신순으로 글의 제목·요약·날짜·태그를 보여준다.
2. **글 상세 페이지** (`/posts/[slug]`) — 본문, 메타(제목·작성일·태그·읽기시간), 코드 블록 신택스 하이라이트, 헤딩 앵커.
3. **태그 페이지** (`/tags/[tag]`) — 해당 태그가 달린 글 목록.
4. **어바웃 페이지** (`/about`) — 운영자 소개와 연락 수단.
5. **다크/라이트 모드** — 시스템 설정을 기본으로 따르고, 명시적 토글을 헤더에 둔다. 선택은 로컬에 저장.
6. **RSS 피드** — `/rss.xml`로 구독자가 갱신을 받아갈 수 있도록 한다.
7. **메타데이터/OG** — 글마다 `<title>`, description, OG image(자동 생성 또는 글별 지정) 처리.

## MVP 제외 사항
- 댓글 시스템 (필요해지면 Giscus 같은 오버레이를 붙인다)
- 본문 검색 (글 수가 적은 동안에는 비용 대비 효과가 낮다)
- 뉴스레터·구독 폼
- 다국어 (1차는 한국어 단일)
- 다중 저자
- 관리용 어드민 화면 (콘텐츠는 파일 기반)

## 디자인 방향
- **미니멀, 텍스트 중심.** 글이 주인공이고 UI는 배경에 가깝다.
- **무채색 + 단일 포인트 색.** 보라/인디고 계열은 쓰지 않는다 (UI_GUIDE 안티패턴 참조).
- **다크/라이트 모두 1급 시민.** 토큰을 양쪽 모두 정의하고 어느 쪽도 후순위로 두지 않는다.
- **본문 가독성 폭** — 데스크톱에서 본문 한 줄이 너무 길지 않도록 좁은 max-width(예: ~`max-w-2xl`).
- **서체** — 1차는 시스템 폰트 + 모노스페이스. 한글 가독성 확보가 필요해지면 Pretendard 검토.

## 콘텐츠 소스
- **파일 기반 MDX.** 글은 `apps/blog/content/posts/*.mdx`에 둔다. frontmatter로 메타(`title`, `date`, `tags`, `summary`, `draft`)를 관리한다.
- 외부 CMS·DB 의존성 없음. 글 발행은 PR 또는 직접 푸시.
- 이미지 등 무거운 자산은 `apps/blog/public/` 또는 별도 CDN 경로(필요 시 결정).

## 비기능 요건
- **빌드 가능성**: `pnpm -F blog build`가 항상 통과해야 한다.
- **접근성**: 시멘틱 HTML, 키보드 포커스 가시성, 충분한 색대비.
- **성능**: 글 목록·상세는 정적 생성(SSG)이 기본. 빌드 시점에 모든 글이 prerendered.
- **SEO**: 페이지별 적절한 메타 태그, sitemap.xml, robots.txt.
