# PRD: Atelier Blog MVP

## 목표

Atelier의 첫 제품은 개인 블로그입니다. 글과 작업물을 안정적으로 게시하고, 이후 다른 프론트엔드 앱을 같은 모노레포 구조 안에서 추가할 수 있는 기반을 만듭니다.

## 사용자

- 작성자: MDX 파일로 글을 작성하고 코드 변경과 함께 배포합니다.
- 방문자: 모바일과 데스크톱에서 글, 프로젝트, 소개 페이지를 읽습니다.

## MVP 범위

1. 홈 페이지: 블로그의 정체성과 주요 진입점을 보여줍니다.
2. 글 목록 페이지: 발행된 글을 최신순으로 보여줍니다.
3. 글 상세 페이지: MDX 본문, 제목, 날짜, 카테고리, 태그, 읽기 시간을 보여줍니다.
4. 프로젝트 페이지: 작업물 목록과 상세 페이지를 제공합니다.
5. 소개 페이지: 작성자와 블로그의 방향을 설명합니다.
6. 아카이브 페이지: 콘텐츠를 한눈에 훑을 수 있게 정리합니다.

## 현재 구현 상태

- 앱은 `apps/blog`에 있습니다.
- 콘텐츠는 `apps/blog/content/blog/*.mdx`에 있습니다.
- 콘텐츠 파싱은 `apps/blog/lib/posts.ts`에서 처리합니다.
- 기본 테스트는 `apps/blog/lib/posts.test.ts`에 있습니다.
- 루트 스크립트는 `pnpm dev`, `pnpm build`, `pnpm lint`, `pnpm typecheck`, `pnpm test`를 제공합니다.

## MVP 제외 항목

- 댓글 시스템
- 검색
- 뉴스레터/구독
- 관리자 화면
- 외부 CMS나 데이터베이스
- 다국어 콘텐츠 관리
- 완성형 CI/CD와 무중단 배포

## 비기능 요구사항

- 접근성: semantic HTML, 키보드 접근, 명확한 focus 상태를 유지합니다.
- 성능: 가능한 페이지는 정적으로 생성합니다.
- SEO: 페이지별 metadata를 점진적으로 채웁니다.
- 유지보수성: 앱 전용 코드는 앱에 두고, 반복되는 코드만 공유 패키지로 분리합니다.
- 배포 가능성: 이후 Docker/Nginx/blue-green 배포 실습이 가능하도록 앱 단위 빌드를 유지합니다.

## 다음 제품 작업

1. 문서와 실제 구조를 계속 동기화합니다.
2. `packages/eslint-config`를 작게 추가합니다.
3. 블로그 앱을 Docker 이미지로 빌드할 수 있게 만듭니다.
4. 로컬 reverse proxy와 blue-green 배포 연습 환경을 추가합니다.
